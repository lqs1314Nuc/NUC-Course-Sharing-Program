# 中北大学课程攻略共享计划

NUC Course Sharing Program

---

## 前言

进入大学之后，我们会陆续接触大量课程。
在这一门门课程开始、进行、结束的过程中，我们往往会不自觉地积累起各种资料与信息。

有些是需要反复搜索才能找到的电子教材；
有些是零散存在于群聊、论坛、网盘中的历年试卷与复习资料；
还有些，是在课程结束后才终于总结出的经验——
比如真正的重点、合理的学习节奏，或是那些“如果早点知道就好了”的细节。

这些信息并非不存在，却往往难以系统获取。
它们依赖运气、依赖人际关系，更多时候只能靠口口相传。

---

## 为什么要做这件事

我曾经尝试尽可能地收集各种课程资料。
但即便如此，一些关键信息的获得，依然常常只是偶然。

更令人遗憾的是：
当一门课真正结束、经验逐渐成型，这些来之不易的总结却往往无处安放，
最终随着时间推移，被遗忘、被覆盖、被彻底丢失。

每一年，都有大量同学在重复相同的试错过程。
这种重复的、孤立的劳动，本不该是常态。

---

## 项目的初衷

本项目希望将这些：

* 隐晦的
* 零散的
* 不确定的
* 依赖口口相传的

课程资料与经验，逐步转化为：

* 公开的
* 可查阅的
* 可积累的
* 可持续完善的

**共享资源。**

我希望，只要是前人已经走过的弯路，
后人就不必再独自重来。

这既是这个项目存在的理由，也是它持续维护的意义。

---

## 内容范围

截至目前，本项目收录内容主要包括（但不限于）：

* 📌 选课与学习建议
* 📘 电子版教材 / 参考书
* 📝 平时作业参考与经验总结
* 📄 历年试卷与复习资料
* 🧠 开卷考试 A4 纸整理
* 📚 各类补充学习材料

内容覆盖中北大学多个专业方向的课程，并将持续扩充。

---

## 为什么选择 GitHub

选择 GitHub 作为平台，主要基于以下考虑：

* 传统群聊与网盘依赖年级与个人维护，难以长期保留
* GitHub 支持清晰的目录结构，便于按课程组织内容
* 每个目录可配套 README，适合记录经验性内容
* Issue 与 Pull Request 机制便于协作、审核与长期维护

GitHub 并非门槛最低的平台，
但它足够稳定，也足够适合一个需要长期存在的共享项目。

## 操作说明

* Fork 本项目
* 上传文件至对应课程目录
* 发起 Pull Request

如果不方便操作 Git，也可以将资料附加在 Issue 中，由维护者协助整理。

请留意项目的文件组织结构，尽量保持清晰、简洁。

---

## 注意事项

* 对教师的评价请统一使用姓名拼音首字母缩写
* 请勿上传当期考试原题
* 教师课件如涉及更新，原则上不建议上传原文件

---

## 不适合上传的内容

以下内容不适合出现在本仓库中：

* 盗版或付费电子书
* 破解版、绿色版商业软件及安装包
* 教师主页或课程平台中明确限制传播的内容（除非获得许可）

如相关资料确有必要，请考虑放置于校内平台，或以链接形式进行指引。

若你认为本仓库中的内容侵犯了你的权益，请联系我们，
我们将在核实后第一时间移除相关文件。
